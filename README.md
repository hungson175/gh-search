# GitHub Project Search (gh-search)

Semantic search MCP server for discovering GitHub repositories using vector embeddings and hybrid search.

## Purpose

**gh-search** helps developers find reference implementations, libraries, and starter projects by searching across 55,000+ high-quality GitHub repositories (1000+ stars) using natural language queries.

## Getting Started

### Prerequisites

- Python 3.11+
- PostgreSQL 17+ with pgvector extension
- API keys: GitHub, Voyage AI, xAI (Grok)

### Installation

```bash
# Clone repository
git clone https://github.com/hungson175/gh-search.git
cd gh-search

# Create virtual environment
uv venv --python 3.11 .venv
source .venv/bin/activate

# Install dependencies
uv pip install -e ".[dev]"

# Configure environment
cp .env.example .env
# Edit .env with your API keys (see Configuration section below)
```

### Configuration

Create `.env` file with required API keys:

```bash
# PostgreSQL
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=github_projects
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres

# Voyage AI (for embeddings)
VOYAGE_API_KEY=pa-your-key-here

# xAI (Grok - for LLM summaries)
XAI_API_KEY=xai-your-key-here
```

### Quick Start

```bash
# Activate virtual environment
source .venv/bin/activate

# Test with 10 repos
python scripts/fetch_readmes.py --limit 10

# Run tests
pytest

# Check code quality
black src/ tests/ && ruff check src/ tests/
```
